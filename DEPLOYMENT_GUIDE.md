## Prerequisites
1. Railway account with a project created at https://railway.app
2. Railway API token from https://railway.app/account/tokens
3. Production environment variables ready

## Step 1: Generate Railway API Token
1. Navigate to https://railway.app/account/tokens
2. Click "Create Token"
3. Copy the token to clipboard
4. Store it securely — you'll use it once

## Step 2: Deploy Backend to Railway

```bash
# Set the API token
export RAILWAY_TOKEN="<your-railway-api-token>"

# Navigate to project directory
cd C:\Pi Dev Ops

# Deploy using Railway CLI
railway up

# Monitor deployment
railway logs -f
```

## Step 3: Configure Production Environment Variables

After deployment, set these in the Railway dashboard or via CLI:

```bash
# Required
export TAO_PASSWORD="<generate-random-32-char-password>"
export ANTHROPIC_API_KEY="<your-anthropic-api-key>"
export GITHUB_TOKEN="<your-github-personal-access-token>"
export LINEAR_API_KEY="lin_api_<your-linear-api-key>"

# Optional (defaults provided)
export TAO_SESSION_TTL="86400"
export TAO_SESSION_SECRET="<auto-generated>"
export TAO_MAX_SESSIONS="3"
export TAO_RATE_LIMIT="30"
```

## Step 4: Verify Production Deployment

```bash
# Get the Railway deployment URL
railway open

# Run smoke tests against production
export RAILWAY_URL="<your-railway-url>"
python scripts/smoke_test.py --url $RAILWAY_URL --password $TAO_PASSWORD
```

## Step 5: Deploy Dashboard to Vercel

```bash
# If not already deployed
cd C:\Pi Dev Ops\dashboard
vercel deploy --prod
```

## Troubleshooting

### "Cannot login in non-interactive mode"
This happens when Railway CLI can't open a browser for login. Use a Railway API token instead (see Step 1).

### "Invalid password" in smoke tests
Ensure `TAO_PASSWORD` is set the same value in both:
- Server environment variables (Railway config)
- Smoke test invocation (`--password` flag)

### Deployment stuck
Check Railway logs: `railway logs` or view in dashboard at https://railway.app
