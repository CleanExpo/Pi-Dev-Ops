## Summary

Ports `@sting8k/pi-vcc` as `app.server.tao_context_vcc`. Algorithmic, transcript-preserving conversation compactor that runs in the hot path with NO LLM calls. Sibling primitive of RA-1966 (`kill_switch`) and pre-requisite leverage point for RA-1970 (`tao-judge` + `tao-loop`) — every loop iteration shrinks the TAO transcript before the next prompt assembly.

## Four deterministic techniques

| Technique | Trigger | Marker |
|---|---|---|
| `tool_dedup` | identical `(role="tool", content)` recurs | `<truncated: same as msg N>` |
| `verbose_truncate` | block >2000 lines or >50 KB | `<elided K lines, B bytes>` |
| `repeat_collapse` | run of identical adjacent messages | `<repeated K times>` |
| `whitespace_normalise` | trailing-ws / blank-line runs / CRLF | (in-place rewrite) |

Pure function. Idempotent on second pass (markers carry stable prefixes). No I/O, no module-level state.

## SDK call-path decision

`_run_claude_via_sdk` accepts a single composite `prompt: str` rather than a structured message list. Inserting compaction here would require either an SDK-call-surface refactor or a parse/render round-trip — both wider than the Wave 1 primitive scope. Per the brief's risk-aware path, the wiring is deferred to RA-1970 once `tao-loop` settles the canonical TAO message representation. A `compact_for_sdk(messages)` helper is provided for future callers, and the deferral is documented inline in `session_sdk.py` and in CLAUDE.md.

## Risk

- `compact()` mutates list-of-blocks assistant content into a single text block when normalisation triggers. Downstream consumers that depend on tool-use block fidelity must NOT call this on assistant messages they need round-trip — the function is for prompt-prefix compaction, not transcript archival.
- Token-budget hint uses bytes/4 as a cl100k_base proxy; it is best-effort and can over-compact narrow-Unicode payloads. Acceptable for the hint role; harness uses real `tiktoken` when available.
- `_run_claude_via_sdk` is unchanged at runtime — only docstring TODO added. Zero risk to RA-1966 kill-switch + the autonomy poller wiring.
