## Tasks

### Task 1: Quality Gate Sweep

Run each command in sequence. Capture full output. Do not stop at first failure — run all four.

```bash
npm run type-check 2>&1
npm run lint 2>&1
npm test -- --passWithNoTests 2>&1
```

For the build, run a type-only dry-run rather than a full Next.js build (avoids API calls
and is faster):

```bash
npx tsc --noEmit 2>&1
```

(This is the same as `type-check` — confirm output matches.)

**Document findings as:**

```
[QUALITY-xx] CRITICAL|HIGH|MEDIUM|LOW
File: <path>:<line>
Issue: <description>
Command: type-check | lint | test
```

Rules:

- TypeScript errors → CRITICAL if in `app/api/` or `lib/`, HIGH if in `components/`, MEDIUM otherwise
- ESLint errors → HIGH; warnings → LOW
- Test failures → CRITICAL
- Zero failures → note "PASS" for that gate

### Task 2: Security Audit

Run each scan and document every finding.

**2a — npm audit:**

```bash
npm audit --audit-level=moderate 2>&1
```

Document each CVE: package, severity, fix available (yes/no).

**2b — Environment variable coverage:**
Read `.env.example`. For each var, grep `app/api/` and `lib/` to verify it is actually used.
Flag any var in `.env.example` that has zero usages → LOW (dead config).
Flag any `process.env.` reference in source that has no corresponding `.env.example` entry → HIGH (undocumented secret).

```bash
grep -r "process\.env\." app/api/ lib/ --include="*.ts" | grep -v "node_modules"
```

**2c — Auth middleware scan:**
Read `middleware.ts`. Verify:

- Public paths list is exhaustive and not over-permissive
- No route under `app/api/` that handles mutations is in the public path list
- `getUserIdFromRequestOrCookies` (or equivalent) is called before any DB query in every POST/PUT/DELETE route

Sample 10 random POST routes from `app/api/` and verify auth guard presence.

**2d — Supabase RLS surface:**
Read `supabase/` directory if present. List any table that has no RLS policy defined.
If no `supabase/` SQL files exist, note "RLS policies not under version control — verify via dashboard".

**Document findings as:**

```
[SECURITY-xx] CRITICAL|HIGH|MEDIUM|LOW
Area: npm-audit | env-coverage | auth-middleware | rls
Issue: <description>
Fix: <suggested action>
```

### Task 3: Package Hygiene

**3a — Outdated packages:**

```bash
npm outdated 2>&1
```

Flag packages more than 2 major versions behind as HIGH. Minor version lag → LOW.

**3b — Unused dependencies:**
Use `depcheck` if available, otherwise manually check: for each package in `package.json`
`dependencies` (not devDependencies), grep for import/require in `app/`, `components/`, `lib/`.
Flag packages with zero source imports as MEDIUM (may be peer dep or dynamic import — note uncertainty).

**3c — Bundle surface estimate:**
List the 10 largest packages by checking `node_modules/<pkg>/` directory size:

```bash
du -sh node_modules/* 2>/dev/null | sort -rh | head -20
```

Flag any package >5MB that is imported in client components (`'use client'`) as HIGH (bundle risk).

**Document findings as:**

```
[PACKAGES-xx] HIGH|MEDIUM|LOW
Package: <name>
Issue: outdated | unused | large-client-bundle
Detail: <version info or import count>
```
