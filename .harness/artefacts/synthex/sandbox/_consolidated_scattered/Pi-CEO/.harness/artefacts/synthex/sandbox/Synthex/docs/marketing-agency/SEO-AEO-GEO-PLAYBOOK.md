# SEO / AEO / GEO Playbook

## Decision

Campaign intelligence should improve clarity, evidence, and answer-readiness. It must not keyword-stuff, fake authority, or create structured data for content that is not visible.

## Principles

- People-first content comes before search mechanics.
- Existing Synthex SEO/GEO/E-E-A-T modules should be reused where practical.
- AI search readiness depends on clear entity relationships, source-backed claims, and accessible content.
- Structured data recommendations must match visible content.

## Campaign Intelligence Outputs

| Output | Purpose |
| --- | --- |
| Buyer intent map | Connect persona pains, jobs, and awareness stage to queries. |
| Query fanout | Generate natural questions and adjacent search intents. |
| FAQ map | Identify questions campaign/landing content can answer honestly. |
| Claim-to-evidence matrix | Ensure factual statements can be substantiated. |
| Entity map | Brand, product, category, location, expert, customer, and source relationships. |
| Structured data recommendations | Suggest schema only for visible page content. |
| AEO/GEO score | Rate answer-readiness and source quality. |

## Awareness Stage Map

| Stage | Query Pattern | Creative Role |
| --- | --- | --- |
| unaware | symptom, frustration, story | Create recognition. |
| problem aware | why, how to fix, what causes | Name the problem clearly. |
| solution aware | options, alternatives, frameworks | Position the solution. |
| product aware | brand, feature, proof, comparison | Show value and evidence. |
| most aware | pricing, demo, implementation, reviews | Reduce risk and prompt action. |

## Rules

- Do not create fake expert attribution.
- Do not cite unverifiable sources.
- Do not claim Google or AI Overview inclusion.
- Do not optimise hidden text.
- Do not create structured data for invisible content.
- Do not use location/service claims the business does not support.

## Acceptance Criteria

- Every campaign has SEO/AEO/GEO notes.
- Every factual claim is linked to evidence or marked unsupported.
- Unsupported claims are blocked from client-ready export.
- Structured data recommendations include visible-content requirements.
