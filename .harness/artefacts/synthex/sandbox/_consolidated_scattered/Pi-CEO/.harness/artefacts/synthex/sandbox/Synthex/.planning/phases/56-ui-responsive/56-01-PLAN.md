# Plan 56-01: Responsive Design & WCAG 2.1 AA — Grid Fixes + Nav Aria

## objective

Fix the three highest-impact responsive and accessibility gaps in the Synthex dashboard:

1. **Unresponsive grid layouts** — Several dashboard pages and components have `grid-cols-3/4/5` with no mobile breakpoints. On a 375 px viewport cells overflow or collapse unreadably. Key files: `calendar/page.tsx`, `listening/page.tsx`, `content/drafts/loading.tsx`, `EngagementPrediction.tsx`, `BrandCalendarView.tsx`, `BrandIdentityCard.tsx`.

2. **`aria-current="page"` missing on active nav links** — `SidebarMenuButton` passes `data-active={isActive}` to the Radix data attribute, but the inner `<Link>` receives no `aria-current`. Screen-reader users cannot identify the current page in the navigation tree (WCAG 4.1.3 / ARIA 1.1).

3. **`MobileMenu.tsx` stale nav** — The mobile drawer lists only 5 links (Dashboard, Content, Analytics, Schedule, Settings) while the sidebar exposes 100+ pages. The mismatch means mobile users on most pages cannot navigate without closing the menu and URL-typing.

Linear: UNI-1635
Phase dir: `.planning/phases/56-ui-responsive/`

## context

### Why `aria-current` not `data-active`

The shadcn `SidebarMenuButton` sets `data-active="true"` when `isActive` prop is truthy. This is a CSS styling hook only — it is not conveyed to assistive technology. The ARIA spec defines `aria-current="page"` as the way to programmatically signal the current-page nav item to screen readers. The fix is one line on the inner `<Link>`.

### Responsive breakpoint strategy

Synthex uses Tailwind default breakpoints: `sm:640px`, `md:768px`, `lg:1024px`. Dashboard pages are always accompanied by a sidebar (240 px collapsed, ~280 px expanded), so effective content width at `md` is ≈490 px. Safe rule:

- 5-col grid → `grid-cols-2 md:grid-cols-5` (each item needs ~90 px minimum)
- 4-col grid → `grid-cols-2 md:grid-cols-4`
- 3-col grid → `grid-cols-2 md:grid-cols-3` (unless cards are very small — then keep 2)

For `grid-cols-2` the minimum card width is fine as-is.

### MobileMenu replacement strategy

The `MobileMenu` is currently only toggled by the `lg:hidden` button in the header (`layout.tsx:647`). The replacement keeps the same drawer shell (slide-in from left, backdrop, ESC key close) but replaces the 5 `menuItems` array with the top-level sidebar groups only (not every sub-item). Groups link to their first child's href. This gives mobile users a manageable ≤10 item list that covers every section.
