## Objective

Build the BullMQ worker that:
1. Picks up `autonomous:execute-task` jobs from the `autonomous-tasks` queue
2. Drives `@anthropic-ai/claude-agent-sdk` `query()` to execute the task
3. Posts progress comments to Linear while the agent runs
4. Posts completion or failure comment, then marks the issue Done
5. Registers the worker in `lib/queue/workers/index.ts`
6. Exposes a `/api/workers/autonomous` route (cron-triggered or manually called) to boot the worker

This plan also resolves the critical open question from research: since `claude` CLI may not be available in Vercel Lambda, the worker is designed to be invoked via a long-running cron route using `waitUntil()` (Vercel Edge) or executed in a separate persistent process. For initial implementation, the worker boots in `instrumentation.ts` (same pattern as existing workers) and uses a fallback graceful failure if the SDK is unavailable.

## Context

### Vercel + Claude Agent SDK: Deployment Strategy

The `@anthropic-ai/claude-agent-sdk` spawns a `claude` subprocess via `child_process`. This requires:
1. The `claude` CLI to be in `PATH` in the execution environment
2. `ANTHROPIC_API_KEY` to be set in the process env
3. A long-lived process (not a short Vercel Lambda request)

For the **initial implementation**:
- Register the worker in `instrumentation.ts` (Vercel calls this on server startup, it persists for the lifecycle of the serverless instance — longer than a single request)
- Add `ANTHROPIC_API_KEY` check in the worker — if missing, fail fast with a descriptive error in the Linear comment
- Wrap the entire `query()` call in a try/catch that catches the "claude not found" error specifically and posts an explanatory comment to Linear

This means the system is functional in local dev (where `claude` is installed) and degrades gracefully in environments where the CLI is absent. The design makes it easy to switch to a persistent worker process (Railway, Fly.io) later.

### Worker Design

- **Concurrency:** 1 (one autonomous task at a time — avoids flooding the repo with parallel agent edits)
- **maxTurns:** 50 (prevents runaway loops)
- **allowedTools:** `['Read', 'Edit', 'Bash', 'Glob', 'Grep']` — full code-editing set
- **Progress comments:** every 10 turns of `assistant` type messages
- **On completion:** post ✅ result comment + update issue state to Done via Linear API
- **On failure/error:** post ❌ comment, re-throw so BullMQ retries (up to 3 attempts per `defaultJobOptions`)
- **On max-turns exceeded:** post ⚠️ comment indicating task needs human breakdown, do NOT retry (mark as completed to avoid infinite retries)

### Marking Issue Done

The Linear SDK requires a workflow state ID to mark done — state IDs are workspace-specific. Pattern:
1. Fetch the issue's team first
2. Query `workflowStates` for that team filtered by `type === 'completed'`
3. Use the first completed state ID

This avoids hardcoding. Cache the result per team to avoid duplicate API calls.

## Tasks

### Task 1: Create `lib/queue/workers/autonomous-task-worker.ts`

```typescript
import { Worker, Job } from 'bullmq';
import { getLinearClient } from '@/lib/linear/client';
import { QUEUE_NAMES } from '@/lib/queue/bull-queue';
import type { AutonomousTaskJobData } from '@/lib/queue/bull-queue';
import { logger } from '@/lib/logger';

// Lazy import — @anthropic-ai/claude-agent-sdk may not be importable everywhere
let queryFn: ((opts: { prompt: string; options: QueryOptions }) => AsyncIterable<SDKMessage>) | null = null;

interface QueryOptions {
  cwd?: string;
  allowedTools?: string[];
  appendSystemPrompt?: string;
  maxTurns?: number;
}

interface SDKMessage {
  type: string;
  subtype?: string;
  result?: string;
  total_cost_usd?: number;
}

async function getQueryFn() {
  if (!queryFn) {
    try {
      const sdk = await import('@anthropic-ai/claude-agent-sdk');
      queryFn = sdk.query;
    } catch (err) {
      logger.error('[autonomous-worker] Failed to import claude-agent-sdk:', err);
      return null;
    }
  }
  return queryFn;
}

// Cache completed state IDs per team to avoid repeated API calls
const completedStateCache = new Map<string, string>();

async function getCompletedStateId(teamId: string): Promise<string | null> {
  if (completedStateCache.has(teamId)) {
    return completedStateCache.get(teamId)!;
  }
  try {
    const linear = getLinearClient();
    const states = await linear.workflowStates({
      filter: { team: { id: { eq: teamId } } },
    });
    const completed = states.nodes.find((s: { type: string }) => s.type === 'completed');
    if (completed) {
      completedStateCache.set(teamId, completed.id);
      return completed.id;
    }
  } catch (err) {
    logger.warn('[autonomous-worker] Could not fetch workflow states:', err);
  }
  return null;
}

async function processAutonomousTask(job: Job<AutonomousTaskJobData>): Promise<void> {
  const { issueId, identifier, title, description } = job.data;
  const linear = getLinearClient();

  logger.info(`[autonomous-worker] Starting task ${identifier}: ${title}`);

  // Verify agent SDK is available
  const query = await getQueryFn();
  if (!query) {
    await linear.createComment({
      issueId,
      body: `## ❌ Agent SDK Unavailable\n\nThe \`@anthropic-ai/claude-agent-sdk\` could not be loaded in this environment. The \`claude\` CLI must be installed and available in PATH.\n\nThis task needs to be run in a persistent worker environment (not Vercel Lambda).`,
    });
    // Don't retry — this is an environment config issue
    return;
  }

  if (!process.env.ANTHROPIC_API_KEY) {
    await linear.createComment({
      issueId,
      body: `## ❌ Missing Configuration\n\n\`ANTHROPIC_API_KEY\` is not set. Configure it in the worker environment.`,
    });
    return;
  }

  const prompt = [
    `You are working on Linear issue ${identifier}: ${title}`,
    description ? `\n\nDescription:\n${description}` : '',
    `\n\nComplete the task described above.`,
    `Create a feature branch named \`fix/${identifier.toLowerCase()}\` before making any changes.`,
    `Run \`npm run type-check\` after making changes to verify no TypeScript errors.`,
    `When done, summarise exactly what files were changed and what was done.`,
  ].join('');

  // Post start comment
  await linear.createComment({
    issueId,
    body: `## 🤖 Autonomous Agent Started\n\nWorking on: **${title}**\n\nI'll post updates as I progress.`,
  });

  let turnCount = 0;
  let maxTurnsExceeded = false;

  try {
    for await (const message of query({
      prompt,
      options: {
        cwd: process.cwd(),
        allowedTools: ['Read', 'Edit', 'Bash', 'Glob', 'Grep'],
        appendSystemPrompt:
          'Never commit directly to main. Always work on a feature branch.',
        maxTurns: 50,
      },
    })) {
      if (message.type === 'assistant') {
        turnCount++;
        // Post progress every 10 turns
        if (turnCount % 10 === 0) {
          await linear.createComment({
            issueId,
            body: `⏳ Agent working... (${turnCount} turns completed)`,
          });
        }
      }

      if (message.type === 'result') {
        if (message.subtype === 'success') {
          const costStr = message.total_cost_usd != null
            ? `$${message.total_cost_usd.toFixed(4)} USD`
            : 'unknown';

          await linear.createComment({
            issueId,
            body: `## ✅ Task Complete\n\n${message.result ?? 'No summary provided.'}\n\n---\n*Turns: ${turnCount} | Cost: ${costStr}*`,
          });

          // Mark issue as Done
          try {
            const issue = await linear.issue(issueId);
            const team = await issue.team;
            if (team) {
              const stateId = await getCompletedStateId(team.id);
              if (stateId) {
                await linear.updateIssue(issueId, { stateId });
                logger.info(`[autonomous-worker] Marked ${identifier} as Done`);
              }
            }
          } catch (stateErr) {
            logger.warn(`[autonomous-worker] Could not update state for ${identifier}:`, stateErr);
          }
        } else if (message.subtype === 'error_max_turns') {
          maxTurnsExceeded = true;
          await linear.createComment({
            issueId,
            body: `## ⚠️ Task Too Complex\n\nThe agent reached the turn limit (50 turns) without completing the task.\n\nPlease break this issue into smaller sub-tasks and re-assign them with the \`autonomous\` label.`,
          });
        } else {
          await linear.createComment({
            issueId,
            body: `## ⚠️ Task Ended\n\nResult type: \`${message.subtype ?? 'unknown'}\`. Human review needed.`,
          });
        }
      }
    }
  } catch (err) {
    const errorMsg = err instanceof Error ? err.message : String(err);

    // Detect "claude not found" specifically
    if (errorMsg.includes('ENOENT') || errorMsg.includes('claude')) {
      await linear.createComment({
        issueId,
        body: `## ❌ Claude CLI Not Found\n\nThe \`claude\` binary is not available in this environment. Ensure the Claude Code CLI is installed.\n\nError: \`${errorMsg}\``,
      });
      return; // Don't retry for missing CLI
    }

    await linear.createComment({
      issueId,
      body: `## ❌ Worker Error\n\nAn error occurred during task execution. Check server logs.\n\nError: \`${errorMsg}\``,
    });

    logger.error(`[autonomous-worker] Error on ${identifier}:`, err);
    throw err; // Let BullMQ retry (up to 3 attempts)
  }

  if (maxTurnsExceeded) return; // Don't retry max-turns failures

  logger.info(`[autonomous-worker] Completed task ${identifier} in ${turnCount} turns`);
}

export function createAutonomousTaskWorker() {
  const worker = new Worker<AutonomousTaskJobData>(
    QUEUE_NAMES.AUTONOMOUS_TASKS,
    processAutonomousTask,
    {
      connection: (() => {
        const redisUrl = process.env.REDIS_URL;
        if (!redisUrl) return { host: 'localhost', port: 6379 };
        try {
          const url = new URL(redisUrl);
          return {
            host: url.hostname,
            port: parseInt(url.port || '6379', 10),
            password: url.password || undefined,
            username: url.username || undefined,
            tls: url.protocol === 'rediss:' ? {} : undefined,
          };
        } catch {
          return { host: 'localhost', port: 6379 };
        }
      })(),
      concurrency: 1, // One autonomous task at a time
    }
  );

  worker.on('error', (err) => {
    logger.error('[autonomous-worker] Worker error:', err);
  });

  worker.on('failed', (job, err) => {
    logger.error(`[autonomous-worker] Job ${job?.id} failed:`, err);
  });

  logger.info('[autonomous-worker] Autonomous task worker started (concurrency: 1)');
  return worker;
}
```

**Note:** The Redis connection logic duplicates what's in `bull-queue.ts`. This is intentional — the worker may eventually move to a separate process that doesn't import the full queue module. If this becomes a maintenance burden, extract `getRedisConnection()` to a shared `lib/queue/connection.ts`.
