# Marketing Agency Recon - 2026-05-15

## Executive Decision

Phase 0 is complete enough to proceed to Phase 1 product architecture. Build the Facebook Video Creator as a new module inside the existing Synthex dashboard and service architecture, not as a detached app.

Do not implement Artlist video generation calls. The verified Artlist Enterprise API surface is music catalogue/search/stream/download, authenticated server-to-server. Treat Artlist Studio/AI video as a manual/product workflow boundary until Artlist publishes an official API for it.

Do not publish to Meta or spend on ads by default. Synthex already has Facebook/social posting code paths, but this product surface must remain draft/export/QA-only unless an explicit server-side `APPROVED_TO_PUBLISH_META_ADS=true` flag and credential review exist.

## Official Docs Verified

### Artlist

- `https://developer.artlist.io/welcome`: Artlist Enterprise API is for music catalogue browsing, streaming, searching, and downloading songs.
- `https://developer.artlist.io/authentication`: OAuth 2.0 client credentials flow with `client_id` and `client_secret`; token endpoint is documented; access tokens expire after 1 hour; client secret must never be exposed client-side.
- `https://developer.artlist.io/general-terms`: standard quota is 100 requests/minute; `/search` is 50 requests/minute; `/download` is 20 requests/minute; 429 uses rate-limit headers.
- `https://developer.artlist.io/use-cases`: use cases focus on pairing music to generative media by mood, pacing, and visual style.
- `https://developer.artlist.io/llms-full.txt`: confirms documented song, artist, album, and downloadable URL OpenAPI surfaces.

### HeyGen

- `https://developers.heygen.com/docs/quick-start`: v3 Video Agent supports prompt-to-video via `POST /v3/video-agents`, async polling via `GET /v3/videos/{video_id}`, and `callback_url`.
- `https://developers.heygen.com/cli`: CLI wraps v3 API, returns structured JSON, supports create, poll, and download workflows.
- `https://developers.heygen.com/mcp/overview`: Remote MCP exposes Video Agent, videos, avatars, voices, lip sync, and translation tools through a hosted MCP endpoint.
- `https://developers.heygen.com/docs/webhooks`: webhook endpoints can be registered and verified with HMAC-SHA256 over raw payloads.
- `https://developers.heygen.com/docs/usage-limits`: pay-as-you-go concurrency limit is 10 concurrent video jobs; 429 returns `Retry-After`; Video Agent supports prompts up to 10,000 characters and supported attachments.

### Meta/Facebook

- `https://developers.facebook.com/docs/marketing-api/` and `https://developers.facebook.com/docs/marketing-api/reference/ad-creative/` are the official Marketing API and creative reference entry points.

## Recommended Phase 1 Direction

1. Produce product specs and policies first; no schema/code until those files are accepted by repo truth.
2. Build around organisation-scoped campaign packages:
   - source data
   - buyer persona
   - board memo
   - concept/script/storyboard
   - evidence pack
   - licence pack
   - QA report
   - export package
   - client handoff
3. Keep Artlist and HeyGen behind provider interfaces with mock/dev providers from day one.
4. Store every factual claim, asset, story, consent item, and creative output with source/evidence metadata.
5. Treat Meta as export QA, not launch automation, until publishing is explicitly approved.
