## Milestone Summary

**Key Decisions:**

- Sentry removed from dependencies — `@sentry/nextjs` v8 webpack plugin hung Lambda cold starts 10+ seconds via `require-in-the-middle` / `import-in-the-middle` hooks; server capture still works via `instrumentation.ts` lazy init
- Health/ping routes have `maxDuration=30` for cold-start survival
- `instrumentation.ts` uses no-throw env validation pattern to prevent build failures
- Invite-only soft launch gate: `NEXT_PUBLIC_INVITE_ONLY_MODE` env var → blocks signup for non-invited users
- OAuth flows refined: PKCE for Supabase, platform connections use `returnTo` parameter
- `getUserIdFromRequestOrCookies()` standardised across all onboarding API routes

**Issues Resolved:**

- Pre-deploy type-check failures from `.next-analyze/` being included in tsconfig
- 1051 false lint errors from build artifact directories not in eslint ignores
- OAuth callback errors not surfacing DB errors correctly
- Onboarding page scroll blocked by `overflow-hidden` on root container
- `getAuthUser` replaced with `getUserIdFromRequestOrCookies` in 6 onboarding routes
- Next.js dynamic slug conflict crashing all API routes (commit `4da97430`)

**Issues Deferred:**

- Stripe test → live key swap requires manual Vercel env update per deployment
- Sentry webpack plugin re-enable pending serverless cold-start fix from Sentry

**Technical Debt:**

- Phases 115-117 executed without formal GSD PLAN.md/SUMMARY.md files (deployment phases were fast-tracked)
- `ignoreBuildErrors: false` enforced post-launch in security hardening session
