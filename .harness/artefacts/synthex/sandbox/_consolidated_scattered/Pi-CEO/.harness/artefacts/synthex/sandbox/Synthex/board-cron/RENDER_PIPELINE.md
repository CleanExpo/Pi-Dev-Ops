# Board Session Render Pipeline — SYN-642

## Directory Structure

```
board-cron/
├── video-scripts/          # JSON scripts output by board session cron
│   └── {date}-{slug}.json  # e.g. 2026-04-07-weekly-review.json
├── video-assets/
│   ├── thumbnails/         # PNG thumbnails generated by @vercel/og
│   └── renders/            # MP4 output from Remotion render
├── episode-logs/           # Episode production logs (JSON)
│   └── {date}-{slug}.log.json
└── remotion/               # Remotion composition project
    └── src/
        └── assets/
            └── sounds/     # Audio assets (see catalogue.json)
```

## Prerequisites (Human Gate — SYN-642)

The following must be configured before the render pipeline can run:

### 1. Remotion CLI
```bash
cd board-cron/remotion
npm install
npx remotion --version  # confirm >=4.x
```

### 2. FFmpeg
Required for Remotion video rendering.
- **Windows**: `winget install Gyan.FFmpeg` or download from https://ffmpeg.org/download.html
- Add to PATH: verify with `ffmpeg -version`

### 3. ElevenLabs API Key
```
ELEVENLABS_API_KEY=sk_...
```
Add to `.env.local` (local dev) and Vercel dashboard (production).

### 4. Audio Assets
All audio files in `remotion/src/assets/sounds/` are `NEEDS_GENERATION`.
See `catalogue.json` for the full list. Generate via Suno/Udio — see SYN-643.

### 5. Voice IDs in rotation-state.json
CEO voice is set. The remaining 8 personas need casting via ElevenLabs.
See SYN-641 for casting instructions.

## Running a Render (once prerequisites are met)

```bash
# 1. Generate board script (automated by cron)
node board-cron/scripts/generate-session.js

# 2. Run Showrunner pipeline
# Use the /showrunner skill with the generated script path
```

| Variable | Purpose |
|---|---|
| `ELEVENLABS_API_KEY` | Voice synthesis |
| `NEXT_PUBLIC_SUPABASE_URL` | Episode logging |
| `SUPABASE_SERVICE_ROLE_KEY` | Episode logging |
