## Phases

### Phase 118: Headless Task-Runner

**Goal:** Build a full autonomous task-runner: Linear webhook → BullMQ queue → Claude Agent SDK worker. Issues labelled `autonomous` that move to "In Progress" are picked up and executed headlessly.
**Depends on:** v8.0 (live platform, existing BullMQ infrastructure, Redis, Supabase)
**Plans:** 3

Plans:
- [x] 118-01: Infrastructure Foundations (LinearClient, HMAC verifier, AUTONOMOUS_TASKS queue, SDKs)
- [x] 118-02: Linear Webhook Receiver (POST /api/webhooks/linear, Zod validation, label filter)
- [x] 118-03: Autonomous Task Worker (BullMQ worker, Claude Agent SDK query(), Linear Done state, progress comments)

**Details:**

The system is a three-layer pipeline:
1. **Webhook layer** (`app/api/webhooks/linear/route.ts`) — Linear POSTs on issue update → HMAC verification → Zod parse → filter `type=Issue, action=update, state.type=started, label=autonomous` → enqueue `AutonomousTaskJobData`
2. **Queue layer** (`lib/queue/bull-queue.ts`) — BullMQ `autonomous-tasks` queue with 3-attempt retry, exponential backoff
3. **Worker layer** (`lib/queue/workers/autonomous-task-worker.ts`) — Picks up jobs, drives `@anthropic-ai/claude-agent-sdk` `query()` with concurrency:1, maxTurns:50, posts Linear comments on progress/completion/failure

Known constraint: `instrumentation.ts` does not auto-start workers (by design — caused cold-start hangs in v8.0). Worker boot strategy for persistent environments is deferred to v10.0.

## Milestone Summary

**Key Decisions:**

- Fail-closed webhook verification: `verifyLinearWebhook()` returns `false` when secret absent — never silently passes
- Lazy LinearClient singleton: throws on missing `LINEAR_API_KEY` at first use, not import time
- Filter on `state.type` not `state.name` — Linear state names vary per team
- Require `autonomous` label — prevents all In Progress issues from triggering automation
- Concurrency:1 in worker — prevents parallel agent edits to the same repo
- Logger type safety: all `logger.warn()` calls wrap caught errors as `{ error: err }`
