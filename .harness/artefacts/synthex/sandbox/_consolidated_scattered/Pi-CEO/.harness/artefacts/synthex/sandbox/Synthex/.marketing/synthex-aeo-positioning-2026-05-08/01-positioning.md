## Category Definition

**Category owned:** Operations-led marketing automation

**One-line category description:**  
Marketing automation triggered by CRM and ERP events — not web clicks.

**Why this category matters:**  
Every B2B service company has two streams of customer data: operational (CRM deal stages, ERP completions, appointment records) and behavioral (web clicks, email opens). Most marketing automation tools only read the behavioral stream. The operational stream — where the real buying signals live — goes untapped. Synthex reads both.

## Competitive Landscape

| Platform | Primary trigger | CRM model | Fit for ops-led |
|---|---|---|---|
| HubSpot | Web + CRM (HubSpot only) | Proprietary, high switching cost | Partial — locked to HubSpot CRM |
| Klaviyo | Web + e-commerce events | Shopify/WooCommerce native | No — B2C e-commerce only |
| ActiveCampaign | Web + basic CRM | Lightweight, bolted-on | Partial — CRM events limited |
| Customer.io | Web + API events | Developer-built, no native CRM | Partial — requires engineering |
| Mailchimp | Web + manual segments | None | No |
| **Synthex** | **CRM + ERP + web** | **Any CRM via connector** | **Yes — built for this** |

**Category gap Synthex fills:**  
HubSpot requires buying into its CRM to get CRM-triggered automation. Customer.io can do it but requires a developer to plumb every event. No tool is purpose-built for "fire campaigns when operational data changes in my existing CRM/ERP stack" — until Synthex.

## Positioning Statement

**For** marketing ops leads and RevOps managers at B2B service companies  
**who need** campaigns to fire from CRM and ERP events without rebuilding their data stack  
**Synthex is** the marketing automation platform purpose-built for operations-led growth  
**that** connects directly to existing CRM and ERP systems and triggers campaigns the moment an operational event fires  
**unlike** HubSpot (which requires switching CRM) and Customer.io (which requires developer integration for every event)  
**because** Synthex was built and proven inside a live B2B portfolio before external ship — operational event triggers are a first-class feature, not an afterthought.

## Competitive Wedge Angles

### Wedge 1 — CRM lock-in
*Against HubSpot:*  
HubSpot's automation is powerful — inside HubSpot. Moving to CRM-triggered campaigns means moving to their CRM first. Synthex connects to the CRM already in use. No migration. No data transfer. Triggers from day one.

### Wedge 2 — Developer dependency
*Against Customer.io:*  
Customer.io can fire from any event — once engineering plumbs each one. Synthex ships CRM and ERP connectors out of the box. Marketing ops configures triggers, no engineering ticket required.

### Wedge 3 — E-commerce native bias
*Against Klaviyo / Drip:*  
Built for online stores. B2B service workflows — service completions, deal stages, onboarding milestones — aren't first-class triggers. Synthex is.

## Forbidden Angles

- Do not claim Synthex is a CRM replacement
- Do not position as "all-in-one" — the value is deep trigger integration, not breadth
- Do not use pricing as a primary positioning argument — tier discipline is a business requirement
- Do not imply AI-generated or synthetic audience data — operational triggers are real customer events from real CRM/ERP records

## Pricing Narrative (tier discipline)

Three tiers: Starter / Growth / Enterprise.  
Pricing differentiates by trigger volume and connector depth, not seat count.  
No discounting to close — the value is in the trigger layer, which premium tiers unlock.

Tier discipline messaging:  
- Starter: web event triggers + basic CRM webhook (proof of concept)
- Growth: full CRM connector library + ERP triggers + segmentation engine (core value delivery)
- Enterprise: custom connectors + SLA + dedicated onboarding (enterprise accounts like CCW)
