## Summary

Two fixes to `app/server/autonomy.py` for the 2026-05-05 prod incident in which the autonomy poller silently died for 16 hours on Railway. Smoke test caught it; manual Railway restart recovered it.

1. **Watchdog around `linear_todo_poller`** — the body of the `while True` loop is now wrapped in `try/except Exception`. A single unhandled exception no longer kills the asyncio task. Each crash logs the full traceback, increments `_poller_iteration_errors`, captures `_last_iteration_error`, records a `poller_iteration_error` event, and fires a one-shot Telegram alert at the 1st and 10th failure (when `TELEGRAM_BOT_TOKEN` + `TELEGRAM_ALERT_CHAT_ID` are set).
2. **Per-team orphan-recovery state lookup** — `_orphan_recovery` no longer hard-codes `"Pi-Dev: Blocked"` (only valid on the RA team workflow). Replaced with a `team_uuid → state_name` map. Defaults: RA → `Pi-Dev: Blocked`, DR-NRPG / SYN / GP / UNI → `Todo`. Override via `TAO_ORPHAN_RECOVERY_STATES` JSON env. Missing-state failures are caught + skip the ticket instead of crashing the routine.

Both fixes are independent — Part A would have caught the visible symptom, Part B would have stopped the underlying crash.

## Risk

- The watchdog catches `Exception`, not `BaseException` — `KeyboardInterrupt` and `SystemExit` still tear down the task. Intentional: those signals are operator-driven, not bugs.
- Telegram alert path uses `urllib` directly with a 5 s timeout. Network hangs cap at 5 s; failures log + continue (never re-raise). No new dependency added.
- Per-team state map ships with hard-coded UUIDs for RA / DR-NRPG / SYN / GP / UNI. New teams are warned about once on first orphan recovery, then default to `Todo`. To customise without a code change, set `TAO_ORPHAN_RECOVERY_STATES`.
- `_run_poller_iteration` and `_process_autonomy_issue` are extracted helpers — net behaviour is identical to the prior inline block. The extraction is what makes the watchdog wrapper possible without indenting half the file.
