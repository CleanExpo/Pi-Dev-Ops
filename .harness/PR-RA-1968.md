## Summary

Ports `@0xkobold/pi-codebase-wiki` as `tao-codebase-wiki`. Self-updating per-directory `WIKI.md` files driven by post-merge git history. Compounds: every merge refreshes context for every TAO session that follows.

## Mechanism

1. Post-merge GitHub Action (`.github/workflows/codebase-wiki.yml`) fires on push-to-main.
2. Action runs `python scripts/run_codebase_wiki.py --since=<previous-sha>`.
3. CLI calls `app.server.tao_codebase_wiki.update_wiki(...)`.
4. `update_wiki` reads `git log <since>..HEAD --name-only`, groups commits by top-level directory, builds a compact summarisation prompt per directory, runs a token-budget guard, calls the SDK (sonnet, role=scribe), writes `<dir>/WIKI.md`.
5. Action commits any WIKI.md changes back with `[skip ci]` to avoid recursion.

## Cost-budget guard

Sonnet 4.6 priced at $3/M input + $15/M output (cited inline). Estimate per directory:

```
prompt_tokens   ≈ len(prompt.utf-8) / 4
completion_tokens = 1500   (default)
cost_estimate = prompt_tokens * 3e-6 + 1500 * 15e-6
```

If `cost_estimate > max_cost_usd`, the run bypasses with `bypass_reason="cost_budget_exceeded"` BEFORE any SDK call.

## Kill-switch integration

`LoopCounter.tick()` (RA-1966) runs at the start of each directory iteration. A `TAO_HARD_STOP_FILE` causes a graceful bypass (`bypass_reason="kill_switch:HARD_STOP"`).
