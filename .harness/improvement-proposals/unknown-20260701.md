The watchdog trusted /health and reported green. Lesson: 'autonomous' is a property of TOPOLOGY, not of how clever the code is. If the always-on path doesn't include every component that needs to run while the human sleeps, the system isn't autonomous — it's a cron job in someone's editor. The anthropic>=0.90 dependency declared in pyproject.toml was missing from the scheduled-task sandbox, so imports blew up, pytest exited 1, and the watchdog escalated CRITICAL to Telegram at 00:38 UTC on 2026-04-12. Tests were actually 46/46 green. A watchdog that cries wolf destroys its own usefulness — after one false CRITICAL alert, every subsequent alert is suspect. Lesson: NEVER escalate CRITICAL from an environment you don't control. Fixes: (a) watchdog uses `pytest --collect-only` not a full run, so imports are exercised without runtime creds, (b) ModuleNotFoundError downgraded to warn with explicit 'sandbox env issue, not a real test failure' message, (c) real test-green truth comes from GH Actions in a known-good environment, not this watchdog.
- ℹ️ **[unknown]** Phill asked about Google's TurboQuant (KV cache quantization) as a memory solution. TurboQuant compresses transformer attention internals on CUDA GPUs to stretch LLM context windows. Pi-CEO's memory problem is different: selecting and condensing relevant business context per per-project agent session. Pi-CEO runs against Anthropic's API (we don't own the model), on CPU-only Railway (no CUDA), and our bottleneck is relevance not raw memory. The right answer is semantic RAG: per-project memory/ folder, retrieval step before session start, weekly summarization of long histories, embedding compression as a later optimization. TurboQuant's rotate-then-quantize trick is actually useful for embedding compression eventually — same intuition, different layer. See TURBOQUANT-ASSESSMENT.md for the full writeup and the four-piece implementation plan.
