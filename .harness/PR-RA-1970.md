## Two siblings

### `tao_judge.judge(goal, workspace, state, *, timeout_s, session_id) -> JudgeVerdict`

Wraps an evaluator-role SDK call (Sonnet 4.6 per RA-1099) with a JSON-only scoring prompt. Mirrors `_decompose_brief`'s JSON-discipline pattern from `orchestrator.py`. `JudgeVerdict.reason` Ôłł {GOAL_MET, INSUFFICIENT_PROGRESS, TESTS_FAIL, TIMEOUT, STILL_WORKING}; `score` Ôłł [0, 1] is the primary scalar. JSON parse failure Ôćĺ STILL_WORKING (continues). `KillSwitchAbort` from any caller in the chain bubbles up ÔÇö never swallowed.

### `tao_loop.run_until_done(goal, workspace, *, max_iters, max_cost_usd, judge_every_n_iters, timeout_per_iter_s, on_event, session_id) -> LoopResult`

One generator step per iter, optional judge call every N iters. `kill_switch.LoopCounter` provides three orthogonal abort axes (`MAX_ITERS`, `MAX_COST`, `HARD_STOP`). `judge_every_n_iters=2` halves judge spend on cost-sensitive sessions. `LoopResult.reason` Ôłł {GOAL_MET, MAX_ITERS, MAX_COST, HARD_STOP, JUDGE_NEVER_SATISFIED}.

## Deviations + reasoning

- `run_until_done` accepts optional `max_iters` / `max_cost_usd` parameters that override env defaults at LoopCounter construction. Documented inline that `LoopCounter` freezes its limits at construction (per RA-1966 design) ÔÇö caller-side overrides are honoured only if explicitly truthy and positive.
- HARD_STOP is checked via `check_hard_stop()` BEFORE the worker step rather than via `counter.tick()`. Reason: an explicit early-out before any expensive SDK call is a kinder operator UX, and the test `test_hard_stop_precedence` asserts the worker is never invoked when the file exists. `counter.tick()` still runs after the worker step and re-checks HARD_STOP as a backstop.

## Risk

- `_quick_pytest` invokes `pytest` as a subprocess inside the workspace each iter ÔÇö bounded to 120 s. Tests mock this out so the test suite stays fast. Production callers can disable by passing a workspace without a `tests/` directory.
- The judge's prompt slice limits last_test_output to 2000 chars and last_diff to 1000 chars ÔÇö enough for headline failures, not enough for full transcript context. The judge is a single-scalar gate, not a code reviewer.
