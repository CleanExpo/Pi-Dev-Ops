# RA-1969 — `tao-context-mode`: summary-index + on-demand expansion

## Summary

Wave-1 port of `context-mode` to TAO. Walks a repo once, summarises every
source file (~200B/file vs raw 5-50KB), and serves the FULL file content
only when callers explicitly `expand(path)`. No LLM calls — pure regex
symbol extraction + first-comment synopsis. Deterministic, sha256-keyed
for invalidation.

## Validation result

**Median 89.22% reduction over vcc baseline** (RA-1967), well above the
40% threshold from the 2026-05-05 board memo.

- Repo indexed: 586 files, 4.4 MB
- vcc baseline: 228,692 tokens (cl100k_base)
- Treatment min/max: 79.0% / 90.0%
- Mean: 87.86%
- 10 hand-crafted questions in `scripts/_context_mode_questions.json`

The Technical Architect's Round-2 challenge held: vcc already gives ~56%
out of the box, so the bar was deliberately set as "additional reduction
beyond vcc, not over raw harness". context-mode clears that bar comfortably
because it filters BEFORE loading rather than compacting AFTER. The two
primitives compose cleanly (index-first, then compact-what-loaded).

## Recommendation

**GO.** Headline metric: 89.22% median additional reduction over vcc
baseline. Disposition logic: PASS (>40% threshold). The build/expand
pattern is composable with the existing vcc compactor.

## Caveats / known issues (out of scope)

- Pre-existing test interaction: when `tests/test_tao_kill_switch.py` runs
  before `tests/test_tao_codebase_wiki.py::test_hard_stop_aborts_before_sdk`
  in the SAME pytest invocation, the wiki test fails because module-level
  state leaks between tests in the kill-switch test file. Reproduces on
  baseline `efa1e86` without my changes — NOT introduced by this PR. File
  a follow-up if the team wants to fix the existing kill_switch test
  isolation.
- Default `TAO_MAX_ITERS=25` trips during a real-repo index scan (one
  tick per file). The validator and CLI runner override to 100,000. If
  production callers use `build_index` against real repos, they must do
  the same. Follow-up: either bump the default for non-LLM callers or
  expose a per-counter override.
