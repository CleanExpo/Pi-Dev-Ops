# Compound Development Loop

Date: 2026-06-16
Status: active operating pattern
Owner: Senior PM

## Decision

Adopt `compound-development-loop` as the named Pi-CEO operating layer for
agentic product development.

The purpose is to move the system closer to a true agentic solution for a
non-engineer operator: the operator should not need to provide an
engineer-complete brief. The system should expand the idea, research adjacent
advances, produce approval artifacts, plan implementation, build, review, and
write the learning back into the 2nd brain and CRM.

## Why

The bloat risk is acceptable because the greater risk is literal execution:
building only the words supplied by the operator, missing adjacent advances,
missing product consequences, and failing to turn discoveries into reusable
system memory.

The loop exists to make each unit of work improve the next one.

## Canonical Loop

1. Strategy anchor
2. Idea expansion
3. Research grounding
4. Requirements or decision brief
5. Implementation plan
6. Build
7. Multi-agent review
8. Compound learning
9. Product pulse

## Active Integration

- `skills/compound-development-loop/SKILL.md` defines the operating contract.
- `src/tao/skills.py` injects the skill for feature, spec, plan, and spike
  intents.
- `scripts/idea_expansion.py` creates expansion packets and CRM-ready approval
  tasks.
- `scripts/process_ideas_inbox.py` expands phone/Telegram ideas before filing
  work.
- `scripts/run_capability_loop.py` continues to scout external AI capability
  opportunities and write 2nd-brain / CRM intake artifacts.

## Guardrails

- Expansion does not equal approval to build every adjacent idea.
- CRM tasks generated by the loop are blocked and approval-required by default.
- External tools remain sandbox-only until licensing, security, maintenance,
  and fit are reviewed.
- Restricted files and production secrets remain governed by `AGENTS.md`.
- The loop can compress for trivial changes, but must not disappear for
  ambiguous, strategic, or cross-system work.

## Done Means

A non-trivial build using this loop should leave behind:

- strategic read
- research lanes or sourced research
- decision/requirements artifact
- implementation plan
- verification evidence
- review ledger
- compound learning target or note

If any item is intentionally skipped, the skip must be named as a waiver or
scope decision.
