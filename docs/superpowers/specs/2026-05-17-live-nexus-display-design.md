# Live Nexus Display — sub-project 3

## Context

Sub-projects 1 and 2 are personal productivity (your Plaud NotePin → wiki → Linear). Sub-project 3 is the **client-facing showcase**: during a live meeting, participants see live notes being generated by Margot as you talk. Polish and reliability matter more than feature depth — it's a sales/credibility tool, not internal tooling.

Plaud's record-then-upload model isn't real-time; this sub-project uses a separate browser-based capture path streaming to AssemblyAI. Persistence flows to a Google Drive folder, NOT local disk (Phill's directive 2026-05-17 — keep meeting data off the PC).

## Decisions (locked in brainstorm)

| Decision | Choice |
|---|---|
| V1 context | **In-person sales calls in your office.** Other 3 contexts (Zoom share, booth, boardroom) are follow-up sprints on the same substrate. |
| Transcription | **AssemblyAI streaming** — sub-second latency, word-by-word, speaker diarization included. ~$0.65/hr after free tier. |
| Display content | **Live transcript (60% width) + AI synthesis sidebar (40%)** showing Topics + Action Items, updated every 30s by Anthropic Haiku 4.5. |
| Branding | **Unite-Group Nexus only** — Gun Metal `#0e1014` + Candy Red `#b30000`. |
| Architecture | **Vercel-hosted SPA + Edge Runtime API routes** for key brokering. Scales to all 4 future contexts without rewrite. |
| Persistence | **Google Drive folder** (`/Brain/Live-Nexus/`). **Drops Supabase from live-display scope.** Per Phill 2026-05-17 — remove data bloat from PC. |
| Auth to Drive | **Google Cloud service account** — credentials in Vercel env var. Folder shared with the service account. |

### Three API routes (all Edge Runtime)

**`POST /api/session`** — mints a 60-second AssemblyAI temp token. Body: `{}`. Returns `{token, ws_url, expires_at}`. Latency budget: <200ms.

**`POST /api/synthesize`** — body `{transcript, current_topics, current_actions}`. Calls Anthropic Messages API with Haiku 4.5, tool-use forced on `update_synthesis` (schema: `{topics: string[], actions: {title, description, priority}[]}`). Returns updated `{topics, actions}`. Latency budget: <2s.

**`POST /api/save`** — body `{meeting_id, started_at, ended_at, transcript, topics, actions, brand}`. Composes markdown with frontmatter (see "Saved file shape" below). Authenticates to Drive via service-account JWT, creates file at `/Brain/Live-Nexus/{YYYY-MM-DD}_{slug}.md` (slug derived from first topic or timestamp). Idempotent on `meeting_id` — re-saving the same meeting updates the existing file. Returns `{drive_url, file_id}`. Latency budget: <5s.

## Data flow

```
T+00:00  Click "Start Meeting"
         ├─ getUserMedia → mic stream
         ├─ POST /api/session → temp token
         ├─ Open AssemblyAI WebSocket, stream PCM-16
         ├─ IndexedDB session begins (auto-saves transcript every 5s)
         └─ Header LIVE dot pulses Candy Red

T+00:03  PartialTranscript: "we've got the new"
         → TranscriptStream renders char-by-char

T+00:05  FinalTranscript: "we've got the new pricing tiers ready"
         → Commits line, starts new line

T+00:30  synthesis-poller fires
         → POST /api/synthesize with last 30s transcript
         → Returns {topics: ["Q2 pricing tiers"], actions: []}
         → Sidebar appends with fade-in

T+01:00…T+60:00  Pattern repeats every 30s

T+60:00  Click "End Meeting"
         ├─ MediaRecorder stops
         ├─ Final /api/synthesize over last chunk
         ├─ POST /api/save → Drive markdown created
         ├─ IndexedDB session marked complete
         └─ Display switches to "Ended" mode
```

### Idempotency / crash recovery

- **Meeting UUID** generated at start; URL becomes `/m/{uuid}`. Refresh resumes the same session from IndexedDB.
- **IndexedDB** persists `{meeting_id, transcript, topics, actions, started_at}` every 5s during recording.
- **Save** is idempotent on `meeting_id` — re-save updates the existing Drive file (not duplicate).
- **Tab close + reopen** within 10 min: Resume prompt restores state; transcript appends from now.

## Error model

| Failure | Detection | Response |
|---|---|---|
| Mic permission denied | `getUserMedia()` reject | Pre-flight blocks Start button. Clear UI: "Open System Settings → Privacy → Microphone." |
| `/api/session` 500 (no AssemblyAI key) | HTTP 500 from edge | Pre-flight tech check fails. Start disabled. |
| AssemblyAI temp token expired mid-meeting (60s expiry) | WebSocket close 4003 | Auto-refetch new token, reopen WebSocket. <3s recovery target. |
| AssemblyAI WebSocket drop (network blip) | `onclose` non-clean | Exponential backoff reconnect (1s, 2s, 4s, 8s). Banner "Connection unstable" after 4 failures. Retries forever. |
| Anthropic synthesis fails | non-2xx from `/api/synthesize` | Silent on first tick. After 3 consecutive: "synthesis paused" pill. Transcript continues unaffected. |
| Browser refresh / accidental tab close | IndexedDB session <10min old on next page mount | "Resume previous meeting?" prompt → restore state, new AssemblyAI WS. |
| Drive `/api/save` fails (auth, quota, network) | non-2xx from edge | Modal: "Retry save" + "Download as file". Download produces same markdown locally as fallback. |
| Drive service-account quota exhausted | HTTP 403 from Drive API | Same as above — Download-as-file fallback. Logs to Vercel; check GCP console. |
| Browser doesn't support PCM-16 MediaRecorder | feature detection on mount | Pre-flight tech check fails. "Use Chrome 120+ or Safari 17+." |
| Long meeting >3h (battery / overheat) | passive | Tab sleeps when unfocused. AssemblyAI disconnects. On resume, transcript preserved; new audio fresh. |

### Visible status surface (RA-1109)

Every error reaches the UI within 2s. No silent failures.

| State | Header dot | Banner | Sidebar |
|---|---|---|---|
| OK | Pulsing Candy Red | hidden | Updating |
| Reconnecting | Pulsing amber | "Reconnecting…" | Frozen |
| Synthesis paused | Solid Candy Red | hidden | "synthesis paused" pill |
| Save failed | Solid grey | Modal with retry + download | Frozen final state |
| Mic blocked | Solid grey | "Mic access required" | Empty |

### What we DON'T handle (by design)

- Multiple simultaneous meetings on same machine — old auto-ends and saves when new started.
- Cross-tab session sync.
- Offline mode — internet required. No queued-save.
- Mic source switching mid-meeting — pick at start; sticks.

## Privacy

- **Audio** never leaves the browser + AssemblyAI's pipeline. AssemblyAI deletes audio after 24h per their data policy.
- **Anthropic call** sends transcript chunks for synthesis. Same data-handling profile as your existing TAO/Margot Anthropic usage.
- **Drive folder** is service-account owned; only you and the service account have access. Public link-share is OFF by default.
- **No raw audio** is stored anywhere persistent.
- **PC has zero meeting data** — IndexedDB is browser-managed and cleared after meeting ends or 10min stale.

## Acceptance criteria

2. Pre-flight tech check passes within 2s (mic, network, browser).

5. Sidebar updates every 30s with Topics and Action Items.

7. Saved markdown matches the spec format exactly (frontmatter + 3 sections).
8. WiFi-drop mid-meeting → Reconnecting banner <2s → recovery <3s after reconnect.
9. Anthropic-fail mid-meeting → synthesis-paused pill → transcript continues; recovers on key restore.
10. Tab close + reopen within 10min → Resume prompt → state restored.
