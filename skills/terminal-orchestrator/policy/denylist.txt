# Terminal Orchestrator — denylist (ratified post-grill 2026-05-26)
#
# Format: one regex per line. Blank lines and lines starting with # are ignored.
# The line preceding each regex (a `## name`) gives the rule name used in audit
# rows. If absent, the regex itself is used as the rule name (truncated to 40c).
#
# Applied in two passes:
#   1. raw-string pre-pass (catches structural / chaining patterns)
#   2. per-token post-pass after shlex.split (catches verbs that survived
#      tokenisation in a way the raw scan missed)
#
# This file is defense-in-depth. The primary validator is shlex tokenisation
# + per-token allowlist (see allowlist.yaml). A command must pass BOTH layers.
