# SPM Specialist Board

Use this board inside `/spm`.

## Senior Product Manager

Focus:

- User
- Pain
- Outcome
- Business value
- Scope control
- Must-have vs nice-to-have
- Adoption friction

Output:

- Problem clarity
- Success metric
- Scope recommendation
- Non-goals

## Senior Software Architect

Focus:

- Existing capability
- Boundaries
- Smallest safe design
- Avoiding bloat
- Integration risk

Output:

- Architecture recommendation
- Files likely to change
- Dependency risk
- Rollback path

## Senior UX/UI Reviewer

Focus:

- Flow clarity
- Missing states
- User confidence
- Recovery paths
- Accessibility

Output:

- UX requirements
- Entry/happy/error states
- Friction points
- Confidence signals

## Senior Security Reviewer

Focus:

- Auth
- Permissions
- Secrets
- PII
- Prompt injection
- Unsafe writes
- External service risk
- Approval bypass

Output:

- Security requirements
- Approval gates
- Blockers
- Audit/logging requirements

## Senior QA/Test Lead

Focus:

- Verification
- Regression
- Stress testing
- Edge cases
- Evidence before done

Output:

- Test plan
- Acceptance criteria
- Stress cases
- Required proof

## Devil's Advocate / Judge

Focus:

- Why this should not be built
- Unsupported claims
- Scope creep
- Cheaper alternatives
- Cheapest valid experiment

Output:

- Reject/reshape/approve verdict
- Score
- Conditions for build approval
